# PanTax-DBG example

This directory contains a compact paired-end toy example for checking that a PanTax-DBG installation can build a reference database and generate species- and strain-level profiles. The example is intentionally small, so it should be used for a smoke test rather than for benchmarking accuracy or speed.

Run the commands below from the repository root:

```bash
cd /path/to/PanTax-DBG
```

## Files

| File or directory | Purpose |
| --- | --- |
| `exampleDB_input_genomes.txt` | Headerless genome manifest used by `pantax-dbg build-custom`. |
| `example_profile_genome_info.txt` | Headered genome metadata table used by `pantax-dbg profile`. |
| `genome/` | Four reference genome FASTA files used in the toy example database. |
| `HRGM2_example.nodes.dmp`, `HRGM2_example.names.dmp` | Taxonomy files for the example identifiers. |
| `my_genome_sizes.ncbi.tsv.gz` | Species genome-size table used during database construction. |
| `read1.fq`, `read2.fq` | Small paired-end toy reads generated from the included genomes. |
| `pantaxdbg_DB/` | Database directory generated by the build command. |
| `profile_res/` | Output directory generated by the profiling command. |

The paths in `exampleDB_input_genomes.txt` and `example_profile_genome_info.txt` are relative to the repository root. `pantaxdbg_DB/` and `profile_res/` are generated during the run and are ignored by Git.

## 1. Build the example database

```bash
mkdir -p example/pantaxdbg_DB

pantax-dbg build-custom \
    -k 31 \
    -w 51 \
    --verbose \
    --input-file example/exampleDB_input_genomes.txt \
    --taxonomy-files example/HRGM2_example.nodes.dmp example/HRGM2_example.names.dmp \
    --db-prefix example/pantaxdbg_DB/pantaxdbg_db \
    --level strain \
    --genome-size-files example/my_genome_sizes.ncbi.tsv.gz \
    -t 16
```

The generated database prefix is:

```text
example/pantaxdbg_DB/pantaxdbg_db
```

## 2. Profile the example reads

```bash
mkdir -p example/profile_res

pantax-dbg profile \
    -r example/read1.fq \
    -r example/read2.fq \
    --db-prefix example/pantaxdbg_DB/pantaxdbg_db \
    --strain-min-reads 5.0 \
    --strain-topk 5 \
    --strain-div 5.0 \
    --r1-large-n 100 \
    --r1-topk-large 10 \
    --r1-min-abundance 1e-7 \
    --ref-info example/example_profile_genome_info.txt \
    --output example/profile_res \
    --species-min-abundance 0.0 \
    --threads 16 \
    -k 31
```

The main outputs are:

```text
example/profile_res/species_abundance.txt
example/profile_res/strain_abundance.txt
example/profile_res/tax_profile.tre
example/profile_res/tax_profile_strain.tre
```

`species_abundance.txt` and `strain_abundance.txt` are the recommended files for downstream abundance analysis. The `.tre` files provide hierarchical taxonomic profiles.
